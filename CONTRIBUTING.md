# 贡献指南

感谢您考虑为 QLLVM 文档做出贡献！您的帮助对我们非常重要。

## 如何贡献

### 1. 报告问题

如果您发现文档中的错误、不准确之处或其他问题，请通过以下方式报告：

- **API参考部分的问题**：在 QLLVM 主仓库中打开 issue
- **其他部分的问题**：在本仓库中打开 issue

### 2. 建议新内容

如果您认为文档中缺少某些内容，或者某些部分可以进一步扩展，请打开一个新的 issue 来提出您的建议。

### 3. 修复问题

您可以查看仓库中的开放 issue，并选择一个适合您的问题来解决。我们建议关注带有 "good first issue" 标签的问题，这些问题通常适合新贡献者。

### 4. 提交 Pull Request

1. Fork 本仓库
2. 创建一个新分支
3. 进行更改
4. 提交您的更改
5. 打开 Pull Request

## 贡献流程

1. **阅读行为准则**：确保您了解并遵守我们的行为准则
2. **选择一个问题**：找到一个您想要解决的问题
3. **进行更改**：在您的分支中进行必要的更改
4. **测试您的更改**：确保您的更改不会破坏现有功能
5. **提交 Pull Request**：提交您的更改并等待审查

## 文档风格指南

- 使用清晰、简洁的语言
- 遵循现有的文档结构和格式
- 确保代码示例正确且可执行
- 提供足够的上下文和解释

## 许可证

通过贡献到本仓库，您同意您的贡献将按照仓库的许可证发布。

## 联系方式

如果您有任何问题或需要帮助，请随时联系我们。
